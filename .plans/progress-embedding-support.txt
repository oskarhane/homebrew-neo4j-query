  var cost for the overwhelmingly common literal-only query-mode path.

  Neo4j accepts as a LIST<FLOAT> parameter for db.index.vector.queryNodes.
- Sequential embed calls (one await per :embed spec) rather than join_all:
  keeps error ordering deterministic and v1 doesn't need the throughput.

Key Decisions:
- OLLAMA_TEST_URL env override mirrors NEO4J_TEST_URI convention — lets CI
  point at a different host if the service port isn't localhost.
- ALL_MINILM_DIMS = 384 kept as a single named constant so the dimension
  assertion is explicit and shared across all 5 tests.
- Round-trip test uses `RETURN size($v) AS n` rather than returning the
  vector itself: Neo4j's HTTP response would dump 384 floats into stdout,
  making assertion brittle on float formatting. size() proves the vector

  "literal types preserved" acceptance criterion — a string "42" would
  also equal 42 under serde_json's loose comparison, so the type check
  is what distinguishes literal-path behaviour from a regression.
- Skip messages printed via eprintln! when ollama/neo4j unavailable so
  `cargo test -- --ignored` in an incomplete env degrades gracefully
  instead of misleadingly failing.
